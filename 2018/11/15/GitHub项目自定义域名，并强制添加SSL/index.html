GitHub项目自定义域名，并强制添加SSL

发表于 2018-11-15

前言

相信很多人都有过使用GitHub Page搭建个人博客的经历，启用后的域名为[你的用户名].github.io,这时我们想要为我们的博客自定义域名只需要在域名服务商添加一条CNAME记录将两者关联进行跳转即可，但如果我们普通的静态文件项目也想利用GitHub page进行展示，同时想要启用自定义域名该怎么办呢？

1、为项目启用Github Page

首先进入项目仓库，进入settings->>GitHub Pages,选择项目所在分支，同时为Custom domain设置准备好的域名。

![设置1](http://wx3.sinaimg.cn/mw690/c3e40a92ly1fx8ualwozpj20l90g2wey.jpg)

这时，使用[你的用户名].github.io/你的项目名可以顺利访问。

2、为自有域名添加A记录

当我们像往常一样使用CNAME解析时，我们会发现，[你的用户名].github.io/你的项目名，这是一个二级域名，而CNAME解析并不支持二级域名，这时候我们ping一下项目的现有地址，得到公网IP，并为它添加A记录。

![设置2](http://wx4.sinaimg.cn/mw690/c3e40a92ly1fx8ui74f94j20vp0cct8x.jpg)

添加A记录后，发现使用自己的域名无法访问该网站，而添加了https头后可以访问，原因是Github16年全面启用了ssl，直接访问器公网IP需要ssl证书，默认情况下使用GitHub Pages的给定域名则支持http和https两种协议，但是如果使用自定义域名的话，则只能通过http://访问。

3.使用CloudFlare 给域名更换解析服务器

实现模式是，用户到CDN服务器的连接为https，而CDN服务器到GithubPage服务器的连接为http，就是在CDN服务器那里加上反向代理。我们登陆Cloudflare输入需要解析的域名选择免费服务，大约一分钟后可以得到一组DNS解析记录，这时移步至域名服务商，腾讯云在管理->>基本信息中修改。

![设置3试](http://wx2.sinaimg.cn/mw690/c3e40a92ly1fx8ut38bgvj20j20cedg1.jpg)

设置完后，我们会发现当我们输入域名时会自动出现小绿锁，可以访问啦~
